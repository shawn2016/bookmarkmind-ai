## 0. 测试目标与原则

### 0.1 一句话目标

> 验证 v3.0 修复「收藏失败」信任危机，补全模型配置与标签入口，且 **不破坏 v2 已交付的 6 个 P0 能力**。

### 0.2 体验宪法（测试不可妥协项）

| # | 原则 | 测试含义 |
|---|------|----------|
| C1 | 收藏优先于智能 | 任何 AI 失败场景下，Chrome 书签栏中必须已有记录 |
| C2 | 一次点击一种结果 | Toast 仅三种结论：全成功 / 部分成功 / 收藏失败；禁止裸错误码 |
| C3 | 渐进式复杂度 | 单击悬浮球 = 快速收藏；标签/备注/文件夹 = 确认弹窗路径 |
| C4 | 删除必确认 | v3.0 不涉及批量删除；v2 清理能力删除流程仍须二次确认 |
| C5 | 三层信息架构 | 本 Sprint 仅验证标签入口与筛选；类型图标属 v3.1 |

### 0.3 不在本计划范围内

- F4 时间轴治理工具栏（v3.1）
- F5 智能分类 v2 / 内容类型图标（v3.1）
- F6 首次使用引导（v3.1）
- Chrome Web Store 上架 Tab 裁剪（Q4，Day 10 另行决策）
- 跨浏览器、付费体系、新 AI 能力

## 2. F1 — 收藏体验重构 · 测试用例

> **核心验收**：无 API Key 收藏成功率 ≥ 99%；AI 失败不阻断收藏；Toast 不出现裸 503。

### 2.1 用例表

| ID | 用例名称 | 前置条件 | 测试步骤 | 期望结果 | 优先级 |
|----|----------|----------|----------|----------|--------|
| F1-01 | 无 API Key 快速收藏 | 配置态 A；任意普通网页 | 1. 单击悬浮球<br>2. 打开 Chrome 书签管理器核对<br>3. 记录响应时间 | ≤ 1s 出现成功 Toast（「已快速收藏」或等价文案）；Chrome 书签栏有该书签；无 AI 网络请求 | P0 |
| F1-02 | 无 Key 连续收藏稳定性 | 配置态 A | 连续收藏 20 个不同 URL（Sprint DoD 要求） | 成功率 ≥ 99%（≥ 20/20 或 19/20 且失败有明确 Chrome API 原因）；每次 Toast 明确 | P0 |
| F1-03 | 有 Key 无 Model 快速收藏 | 配置态 C | 1. 单击悬浮球<br>2. DevTools Network 观察 | ≤ 1s 成功 Toast；书签已创建；**不调用** classify API；不出现 503 | P0 |
| F1-04 | AI 分类成功（全成功 Toast） | 配置态 B；AI 服务正常 | 1. 单击悬浮球<br>2. 等待分类完成（可选二次 Toast） | 书签已创建；Toast 含文件夹名，如「已收藏，归入「开发」」；Chrome 书签位置与分类一致 | P0 |
| F1-05 | AI 返回 503（部分成功） | 配置态 B；Mock AI 503 | 1. 单击悬浮球<br>2. 立即检查 Chrome 书签<br>3. 阅读 Toast 文案 | 书签 **已存在**；Toast 为「已收藏（智能分类暂不可用）」或 PRD 定稿等价文案；**不出现**裸「503」 | P0 |
| F1-06 | AI 超时（部分成功） | 配置态 B；Mock 35s 延迟 | 1. 单击悬浮球并计时<br>2. 2s 内检查 Toast 与书签 | 收藏 ≤ 2s 完成；Toast 部分成功；后台分类失败不 crash；书签仍在 | P0 |
| F1-07 | 关闭自动分类后收藏 | 配置态 D | 1. 设置页关闭「自动分类」<br>2. 单击悬浮球<br>3. Network 验证 | 无 classify 请求；Toast「已快速收藏」；书签正常 | P0 |
| F1-08 | 自动分类开关即时生效 | 配置态 B | 1. 开启自动分类并收藏一页<br>2. 关闭开关不重启扩展<br>3. 再收藏一页 | 第二步有 AI 请求；第三步无 AI 请求 | P0 |
| F1-09 | Chrome 书签写入失败 | 模拟权限异常或书签栏满（若可复现） | 单击悬浮球 | Toast「收藏失败：{可读中文原因}」；不出现技术栈/错误码 | P1 |

| F1-11 | 错误码人性化 — 网络错误 | 断网或 ECONNREFUSED | 收藏（自动分类开启） | Toast 含「网络」或「AI 服务暂时不可用」类文案；若书签已写入则部分成功 | P0 |
| F1-12 | SW 被终止后收藏 | 配置态 B | 1. 收藏一页<br>2. `chrome://serviceworker-internals` 或扩展页停止 SW<br>3. 再收藏 | 书签仍写入；扩展不 crash；必要时提示「扩展未响应，请刷新页面」而非误报收藏失败 | P1 |
| F1-13 | Popup 收藏反馈 | 禁用 content script 的页面或直接用 Popup | 通过 Popup 收藏当前 Tab | 有明确成功/失败反馈（Toast 或 inline）；行为与 F1 原则一致 | P1 |

| F1-16 | 全成功 vs 部分成功 vs 失败 互斥 | 分别构造三种场景 | 对比 Toast 文案 | 三种 Toast 语义互不混淆；用户不会把部分成功当完全失败 | P0 |
| F1-17 | Modal 收藏 bookmarkId 修复 | 配置态 B；面板打开确认弹窗 | 1. 面板「收藏此页」→ 弹窗<br>2. 选标签+备注保存<br>3. 验证书签与标签 | 标签、备注写入成功（依赖 F1-3 / F3 联调） | P0 |
| F1-18 | 异步分类完成通知（若有） | 配置态 B；慢速 AI | 收藏后等待分类完成 | 可选：分类成功后 Toast 更新或列表刷新；不影响已收藏状态 | P2 |

### 2.2 F1 专项检查项

- [ ] 源码/产物全文搜索：用户可见文案中无裸 `503`、`500`、`ECONNREFUSED`
- [ ] `BOOKMARK_CREATE` 响应在书签写入成功后始终 `success: true`
