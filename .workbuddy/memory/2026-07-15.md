## Bug 修复（3 个交互问题）

用户测试后发现三个问题，已全部修复：

### 1. 悬浮球点击无反应
- **根因**：`useDrag.ts` 的 `onPointerDown` 调用了 `e.preventDefault()`，Chrome 下会抑制后续 `click` 事件
- **修复**：移除 `preventDefault()`，改用 CSS `userSelect: 'none'` 防止拖拽时选中文字

### 2. 悬浮球和操作栏闪退
- **根因**：两个问题叠加：
  1. 悬浮球（right:8px, width:48px）和操作栏（right:60px）之间有 4px 间隙，鼠标穿越时触发 mouseleave
  2. 操作栏的 `handleMouseEnter` 只清了自己的 timer，没重置 `ballState`，500ms 后 ballState 变 collapsed → 操作栏被卸载
- **修复**：
  - 操作栏 `handleMouseEnter` 增加 `setBallState('hover')` 重置状态
  - 新增透明桥接 div 覆盖 4px 间隙，鼠标穿越不中断 hover
  - 操作栏偏移从 `BALL_SIZE + 12` 改为 `BALL_SIZE + 8`

### 3. 设置页无法跳转
- **根因**：`OPEN_OPTIONS_PAGE` 消息在 SW 中无 handler，消息类型未定义
- **修复**：
  - `shared/types/index.ts` 新增 `OPEN_OPTIONS_PAGE` 消息类型
  - `background/index.ts` 新增 handler 调用 `chrome.runtime.openOptionsPage()`
  - 迷你操作栏"设置"按钮直接发 `OPEN_OPTIONS_PAGE` 消息
  - 面板内"打开设置页面"按钮增加 `chrome.runtime.openOptionsPage()` 直接调用 fallback

### 第五轮修复（20:44）— SW document is not defined
1. **根因**：`createProvider()` 使用动态 `import()` 加载 AI provider，Vite 自动注入 `__vitePreload` polyfill，该 polyfill 使用 `document.getElementsByTagName` / `document.createElement` / `window.dispatchEvent` — Service Worker 环境没有 `document` 和 `window`，导致 `createProvider` 一调就报 `document is not defined`
2. **修复**：`provider.ts` 将动态 `import()` 改为静态 `import`，`createProvider` 从 `async` 改为同步函数。Provider 代码（openai/anthropic/custom）全部内联到 SW bundle，不再生成单独 chunk
3. **额外修复**：`vite.config.ts` 新增 `build.modulePreload: false`（双重保险）
4. **验证**：构建后 SW bundle (`index.ts-*.js`) 中 `document` 引用数 = 0，`window` 引用数 = 0，不再有 `openai-*.js` / `anthropic-*.js` / `custom-*.js` 单独 chunk
5. **影响**：SW bundle 从 16.6KB 增至 22.0KB（provider 代码内联），但消除了 SW 环境兼容性问题
