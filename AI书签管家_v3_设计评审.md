## 0. 评审摘要

v3 设计重心是**路径清晰、结果透明、治理可反悔**，而非堆功能。近期用户反馈（503 误判、模型选择断裂、标签找不到、时间轴只读）本质都是**结果不透明与入口不可见**，设计需从产品原则出发修复，而非叠加新界面。

需对齐 UI 规范：主色 `--bm-primary-500`、语义色 Success/Warning/Error/Info、8 点间距网格，以及悬浮球（48px）、Toast（max 320px）、Modal 等既有组件规格。v3.0 聚焦 F1–F3 三项 P0；v3.1 交付 F4–F6，上架策略见 Q4。

## 1. F1–F6 UX 流程

### F1 收藏体验重构（P0）

1. 用户单击悬浮球（48×48px）→ 200ms 内有点击反馈（`scale(0.95)`）
2. 系统同步写入 Chrome 书签，完成前显示 Spinner、禁止重复点击
3. AI 分类异步执行，失败不阻断
4. Toast 给出唯一结论：全成功 / 部分成功 / 失败（见 §4）
5. 可选：悬停操作栏 →「收藏此页」打开确认弹窗（F3 路径）

### F2 模型配置引导闭环（P0）

1. 设置页填写 API Key → 点击「测试连接」
2. 成功后模型字段自动变为下拉 Select，下方 Info 条：「已检测到 N 个模型，请选择默认」
3. 选模型并保存 → Success Toast
4. 未选模型时 AI 入口置灰「请先选择模型」，**收藏不受影响**
5. 切换服务商时清空旧列表

### F3 标签体系产品化（P0）

1. 书签 Tab 顶栏：Tag Chips +「管理标签」文字按钮（首屏可见）
2. 点击管理 → 标签 CRUD（设置页 TagManager 或 Drawer）
3. 面板底部「收藏此页」→ 确认弹窗（文件夹 + 标签 + AI 建议 2–3 个）
4. 收藏后按标签筛选可找到该书签
5. 书签详情可编辑标签

### F4 时间轴治理工具栏（P1）

1. 时间轴 Tab → 日期筛选下方「快捷治理」条
2. 切换分组：时间 / 站点 / 内容类型（纯视图）
3. 「归档 N 年前」→ 确认 Modal，默认高亮「移入归档」
4. 「删除 N 年前」→ 二次确认 Modal：精确数量 + 四按钮（取消 / 导出 CSV / 移入归档 / 确认删除）
5. 「仅看无标签」→ 筛选便于补标

### F5 智能分类 v2（P1）

1. 列表行：标题 + 内容类型图标 + 1–2 标签（不堆文件夹名）
2. 一键整理预览「建议 N 个移入历史归档」，不自动删除
3. GitHub 用类型图标 + 标签 `github`，不单独建文件夹
4. 时间轴 Info 条：「可归档 N 个」

### F6 首次使用引导（P1）

1. Step1 聚光灯悬浮球：「点一下就能收藏，不用先配 AI」
2. Step2 高亮设置入口：「配 AI 后可分类和对话搜索」→ 去设置 / 稍后
3. Step3 高亮标签入口：「打标签比建文件夹更好找」
4. 全程可跳过；设置页可重新观看

## 2. 关键界面线框描述

### 2.1 悬浮球 + Toast（F1）

页面右下角：48×48 悬浮球（默认 30% 透明度）。Toast 出现在球体上方，最大宽 320px。

- **Success**（绿）：「已收藏，归入「开发」」— 3s 消失
- **Warning**（黄）：「已收藏（智能分类暂不可用）」— 5s，可关闭，禁止出现 503
- **Error**（红）：「收藏失败：{人话原因}」— 5s，含重试
- 进行中：球内 Spinner，延迟 Toast 至写入完成

### 2.2 标签入口（F3）

书签 Tab 顶栏：`[全部] [react] [待读] ... [管理标签 ›]`，Chips 横滑，管理标签为 Primary 文字按钮（点击区 ≥36px）。下方左栏文件夹 + 右栏列表（类型图标 + 标签）。收藏弹窗含标签输入 + AI 建议 Pills（`--bm-info-50` 底）。

### 2.3 时间轴治理条（F4）

筛选区下方 44px 条，灰底：`分组[时间▾] | [归档1年前] [删除1年前] [仅看无标签]`。归档为 Secondary，删除为 Danger-outline（非 Primary）。320px 窄屏改两行。

删除 Modal（400px）：「将删除 127 个书签（2024-07-16 之前）」+ 回收站说明。按钮序：`[取消] [导出CSV] [移入归档★Primary] [确认删除 Danger]`。大批量（>50）删除需二次确认。

### 2.4 首次引导（F6）

3 步 Coach Mark：半透明遮罩 + 聚光灯镂空。卡片宽 280px，进度 1/3–3/3，按钮「下一步 / 跳过」。Step2 可选「去设置」。

## 3. 信息架构与导航

**三层体系不混用**：

| 维度 | 用户问题 | 界面位置 |
|------|----------|----------|
| 内容类型 | 这是什么？ | 列表行首图标 |
| 文件夹 | 放在哪？ | 书签 Tab 左栏 |
| 标签 | 怎么搜？ | 顶栏 Chips + 行尾 |

**Tab 建议**：v3.0 保留对话 / 书签 / 清理；时间轴待 F4 就绪再开放（Q4）。对话 Tab 负责「找」（自然语言搜索）；书签 Tab 负责「管」（文件夹 + 标签）；清理 Tab 承接 v2 能力，不扩 scope。

**设置全页 IA**：AI 配置 → 标签管理 → 收藏与分类 → 悬浮球 → 数据管理（回收站/导出）→ 帮助与引导。模型引导闭环（F2）与标签管理（F3）必须在设置页有明确导航项，避免「测试成功不知道下一步」。

**收藏路径**：单击 = 快速收藏（v3.0 默认）；面板底部「收藏此页」= 完整确认弹窗；双击开弹窗放 P1（见 Q1）。渐进式复杂度：默认一键，高级能力通过面板与弹窗展开。

## 4. 可访问性与微文案

**无障碍基线**（对齐规范 §17）：Toast 用 `role="alert"`；Modal 焦点 trap；按钮/Chip 最小 36px；尊重 `prefers-reduced-motion`。

**Toast 分级**：

| 级别 | 样式 | 场景 | 文案 |
|------|------|------|------|
| L1 | Success | 收藏+AI 成功 | 已收藏，归入「{夹}」 |
| L2 | Warning | 收藏成/AI 败 | 已收藏（智能分类暂不可用） |
| L3 | Error | 写入失败 | 收藏失败：{人话原因} |
| L4 | Info | 模型已加载 | 已检测到 N 个模型，请选择 |
| L5 | Modal | 批量删除 | 不用 Toast |

**禁止用户可见**：503、401、堆栈。映射：503→智能分类暂不可用；401→API Key 无效。

**关键微文案**：无 AI——「不配置也能收藏；配置后可智能分类」；空标签——「打标签比建文件夹更好找」；归档——「移入历史归档，可随时移回」。

## 5. 设计风险与 Q1–Q4 建议

| # | 问题 | UX 建议 |
|---|------|---------|
| Q1 | 单击改弹窗？ | **保持快速收藏**；v3.1 加双击开弹窗；设置页可配点击行为 |
| Q2 | history 权限？ | **v3 不做**；治理仅依收藏时间与元数据 |
| Q3 | 历史归档保留？ | **是**；📦+锁定 Badge，可重命名不可删，整理预览不可合并 |
| Q4 | 隐藏未完成 Tab？ | **是**；隐藏时间轴；书签 Tab 必须含标签入口 |

**其他风险**：

- Toast 堆叠致忽视 → 收藏场景同时最多 1 条，全局堆叠上限 3（规范已有）
- 删除误触 → Modal 内 Primary 固定「移入归档」，删除用 Danger 且置最右
- 类型识别不准 → 图标 hover 显示类型名；P2 再做手动修改
- 引导跳过率高 → 书签详情首次编辑标签时增加 inline tip 补强 F6
