## 0. Sprint 目标与原则

### 0.1 一句话目标

> 修复「收藏失败」信任危机，补全模型配置与标签入口，让 v2 已有能力真正被用户发现和使用。

### 0.2 体验宪法（Sprint 内不可妥协）

1. **收藏优先于智能** — `chrome.bookmarks.create` 成功是第一性原理；AI 失败不得阻断收藏。
2. **一次点击一种结果** — Toast 必须给出明确结论（成功 / 部分成功 / 失败），禁止裸错误码。
3. **渐进式复杂度** — 单击快速收藏；标签、备注、文件夹选择走「更多收藏路径」。
4. **本 Sprint 最多 3 个 P0 主题** — F1、F2、F3，其余一律不进 v3.0。

### 0.3 团队角色建议

| 角色 | 职责 |
|------|------|
| 产品 | 文案验收、开放问题拍板、DoD 签字 |

### F1 — 收藏体验重构

#### S-001 收藏链路现状审计

| 项 | 标准 |
|----|------|
| 产出 | 一页链路图：`FloatingBall 点击` → `BOOKMARK_CREATE` → `createBookmark` → `autoClassifyOnCreate` |
| 验收 | 明确列出：哪些环节可能向用户暴露 AI 错误；503 出现位置与触发条件 |
| DoD | 产品与研发共同签字确认问题清单；无遗漏的「收藏失败」路径 |

#### S-002 收藏与 AI 分类解耦

| 项 | 标准 |
|----|------|
| 行为 | Step 1 `chrome.bookmarks.create` 必须先完成；Step 2 AI 分类异步且 `try/catch` 静默 |
| 验收 | AI 服务不可用 / 超时 / 503 时，Chrome 书签栏中仍可见新书签 |
| DoD | 3 种 AI 失败场景（无 Key、503、超时）手动测试均 PASS；`success: true` 在书签写入成功后始终返回 |

#### S-003 Toast 三级文案体系

| 项 | 标准 |
|----|------|
| 行为 | 全成功 / 收藏成功+AI 失败 / 收藏失败，三种 Toast 文案互不混淆 |
| 文案 | 全成功：「已收藏，归入「{文件夹}」」；部分成功：「已收藏（智能分类暂不可用）」；失败：「收藏失败：{原因}」 |
| DoD | 产品验收三种文案；用户无法从 Toast 误判「扩展坏了」 |

#### S-004 错误码人性化映射

| 项 | 标准 |
|----|------|
| 行为 | UI 层不出现裸 `503`、`500`、`ECONNREFUSED` 等 |
| 映射 | 503 →「AI 服务暂时不可用」；网络错误 →「网络连接失败」；Chrome API 错误 → 可读中文 |
| DoD | 全文搜索产物与源码，零处裸错误码暴露给用户 |

#### S-005 关闭自动分类开关验收

| 项 | 标准 |
|----|------|
| 行为 | 设置页可一键关闭「自动分类」；关闭后收藏路径零 AI 调用 |
| 验收 | DevTools Network 验证关闭后无 classify 请求 |
| DoD | 开关默认状态与 PRD 一致；切换后立即生效，无需重启扩展 |

#### S-006 无 API Key 收藏成功率验证

| 项 | 标准 |
|----|------|
| 指标 | 无 API Key 场景下 20 次连续收藏，成功率 ≥ 99%（仅 Chrome API 限制可失败） |
| 环境 | 全新安装 / 清空 storage 两种状态均测 |
| DoD | QA 记录测试表格；失败项有 root cause 且已修复或登记 Known Issue |

### F2 — 模型配置引导闭环

#### S-007 测试连接后模型下拉引导

| 项 | 标准 |
|----|------|
| 行为 | 测试成功 → 5 秒内模型字段变为下拉选择；下方显示「已检测到 N 个模型，请选择一个作为默认」 |
| 验收 | OpenAI / Anthropic / Custom 三种 Provider 各测一次 |
| DoD | 新用户从「填 Key」到「看到可选模型」无需额外操作 |

#### S-008 切换服务商清空旧模型列表

| 项 | 标准 |
|----|------|
| 行为 | 切换 Provider 时清空 `availableModels`，避免展示上一服务商模型 |
| DoD | 切换后下拉仅显示新 Provider 模型或空态提示 |

#### S-009 AI 功能入口「请先选择模型」态

| 项 | 标准 |
|----|------|
| 行为 | 已配 Key 但未选模型时，对话 / 一键整理等 AI 入口显示「请先选择模型」 |
| 约束 | **不影响收藏**；仅禁用或引导 AI 功能 |
| DoD | 3 个 AI 入口（对话、整理、分类）状态一致；点击可跳转设置页模型字段 |

#### S-010 帮助文案：AI 可选

| 项 | 标准 |
|----|------|
| 文案 | 设置页明确区分：「不配置 AI = 可以收藏；配置 AI = 可以智能分类和对话」 |
| 位置 | 模型配置区 Onboarding / Callout 组件 |
| DoD | 产品书面确认文案；无「必须配置 AI 才能使用」的误导表述 |

| 项 | 标准 |
|----|------|

| 约束 | 单击悬浮球仍为快速收藏，**不**打开弹窗 |
| DoD | 两条收藏路径（快速 / 高级）行为符合 PRD 分级表 |

#### S-015 带标签收藏后筛选可达

| 项 | 标准 |

| DoD | 至少测试 AND / OR 两种筛选模式；新建标签立即可筛 |

#### S-016 AI 建议标签规则验收

| 项 | 标准 |
|----|------|
| 规则 | AI 可用时建议 2～3 个标签；不自动创建超过 5 字的中文标签名 |
| 约束 | 用户不选标签不强制 |
| DoD | 10 个不同 URL 测试，建议数与字数规则符合预期 |

#### S-017 标签入口可发现性走查

| 项 | 标准 |
|----|------|

| DoD | 走查记录归档；阻塞项在 Day 9 前关闭 |

## 6. 本 Sprint 不做什么

> 以下条目 **明确排除在 v3.0 Sprint 之外**。若 Standup 出现相关需求，一律记入 v3.1 Backlog，不插队。

### 6.1 PRD 已声明的非目标

| 不做 | 原因 |
|------|------|
| 新 AI 能力（图谱、Agent、多轮规划） | v3 聚焦体验，非能力扩张 |
| 跨浏览器同步 | 范围外 |
| 付费体系 | 范围外 |
| 全自动删书签 | 违反「删除必确认」宪法 |
| v2 底层架构重构 | 除非 F1 修复所必需的最小改动 |

### 6.2 顺延至 v3.1 的功能

| 功能 | PRD 编号 | 说明 |
|------|----------|------|
| 时间轴快捷治理（归档/删除 N 年前） | F4 | P1，RICE 50 |
| 智能分类 v2（内容类型图标、历史归档策略） | F5 | P1，RICE 24，Effort L |
| 首次使用 3 步引导 | F6 | P1，可依赖 v3.0 数据决策是否提前 |
| 悬浮球双击/长按打开高级收藏弹窗 | F3 P1 | Q1 决议后留 v3.1 |
| 用户手动修改内容类型 | F5 P2 | 类型识别不准时的补救 |

### 6.3 本 Sprint 不做的体验/工程项

| 不做 | 说明 |
|------|------|
| Tab 裁剪（Q4 上架包） | 除非 Day 10 明确上架，否则不排 |
| 基于浏览历史的「久未访问」 | Q2 决议：v4 再评估 |
| 标签/文件夹/类型的完整用户教育动画 | 仅保证入口与基础文案 |
| 性能优化（5000+ 书签时间轴等） | 非 v3.0 阻塞项 |
| 国际化 / 多语言 | 当前仅中文 |
| 单元测试覆盖率提升 | 除非覆盖 F1 关键路径 |

### 6.4 范围膨胀防护

- **Sprint 内新增需求 &lt; 0.5d**：产品评估后可替换等 Effort 的 Backlog 项
- **Sprint 内新增需求 ≥ 0.5d**：必须延期至 v3.1，或显式 Swap 掉已有 Story
- **Bug 优先级**：P0（收藏失败/数据丢失）&gt; P1（引导/文案）&gt; P2（视觉）&gt; P3（Nice-to-have）
